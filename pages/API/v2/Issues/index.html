Issues are a way for Content Owners and Translation Resources to communicate about translation work. Each Issue is attached to one unique source string in a Smartling project. The issue is either a Source string issue or a Translation issue. If an issue is a translation issue then it is also associated with a specific target translation language. An issue is either Open or Closed and can have any number of user comments attached. Source Issues are visible to all resources in a project who can see the associated string. Translation Issues are visible only to resources who work on the language. Automatic email notifications are generated through the lifecycle of issues, depending on individual user’s profile settings.

For more information about how Issues work in the Smartling Dashboard, see our user documentation.

Smartling’s Issues API provides methods for Creating issues, changing the state of an issue, commenting on an issue and requesting detailed information and reports on issues and comments.

Authentication

Issues API uses OAuth 2.0 authentication. Every request must include an Outh token in the header. See Authentication for more information.

Parameters

GET requests require no parameters except path variables to identify Projects, Issues and Comments. For POST requests, parameters should be passed in JSON format rather than multi-part form.

Parameter Dictionary

The following parameters may be used in Issues API requests and responses. See the pages for each method to find which parameters are available for that method.

Parameters

projectId

Unique identifier for a Smartling project containing an issue. Project ID can be found on the API page of the Smartling Dashboard.

issueUid

Unique identifier for an issue within a Smartling project.

issueTypeCode & issueSubType Code

Identifies the issue type. issueTypeCode is either SOURCE or TRANSLATION. Each have their own set of issueSubType values:

  • SOURCE
    • CLARIFICATION
    • MISSPELLING
  • TRANSLATION
    • POOR_TRANSLATION
    • DOES_NOT_FIT_SPACE
    • PLACEHOLDER_ISSUE
    • GENERAL_ISSUE
    • REVIEW_TRANSLATION

hashcode/hashcodes

Unique identifier for a string in Smartling. Used by the Issues API to attach an issue to a string. Depending on the request, may be a single value or an array.

localeId/localeIds

An identifying code for a language in Smartling. See the full list of locale codes. Depending on the request, may be a single value or an array.

issueStateCode

Either OPENED or RESOLVED.

issueText

Unformatted plain text field explaining an issue.a

commentText

Unformatted plain text of a comment on an issue.

createdDateBefore

Searches for Issues created before a given date. See Date Format for details.

createdDateAfter

Searches for Issues created after a given date. See Date Format for details.

offset / limit

Determines the number of results returned and index offset for large result sets. See Limit and Offset

resolvedByUserUid

Unique identifier for the user who closed the issue.

reportedbyUserUid

Unique identifier for the user who opened the issue.

active

Boolean - true if the string the issue is associated with is active in the Smartling Project, false if the string is inactive.

createdDate

Date an issue was created.

resolvedDate

Date an issue was most recently closed.

sourceLocale

The Smartling locale code for the source language of a string with an issue

targetLocale

The Smartling locale code for the target language for a string with a translation issue
